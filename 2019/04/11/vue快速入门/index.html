vue快速入门

发表于 2019-04-11 | 更新于 2019-07-01

首先需要node环境
Vue-cli是Vue.js的脚手架工具,用于自动生成vue.js+webpack的项目模板,主要作用：目录结构、本地调试、代码部署、热加载、单元测试

  1. 安装vue-cli cnpm install --global vue-cli
    安装之后可以使用vue list查看常用的项目模板名称

  1. 初始化项目/安装webpack项目模板,项目名称不能大写 vue init webpack my-project,
    Project name :项目名称 ,如果不需要更改直接回车就可以了。注意：这里不能使用大写。
    Project description:项目描述,默认为A Vue.js project,直接回车,不用编写。
    Author：作者,如果你有配置git,他会读取.ssh文件中的user。
    Install vue-router? 是否安装vue的路由插件,Y代表安装,N无需安装,下面的命令也是一样的。
    Use ESLint to lint your code? 是否用ESLint来限制你的代码错误和风格
    setup unit tests with Karma + Mocha? 是否需要安装单元测试工具Karma+Mocha。
    Setup e2e tests with Nightwatch?是否安装e2e来进行用户行为模拟测试。

  2. 进入项目 cd my-project

  3. 安装依赖 npm install
  4. 启动项目 npm run dev
    npm run dev就是去执行webpack-dev-server --inline --progress --config build/webpack.dev.conf.js这句话的意思是利用 webpack-dev-server 读取 webpack.dev.conf.js 信息并启动一个本地服务器

项目目录结构:

.babelrc babel配置文件，把我们ES2105的代码通过它编译成ES5的。
.editorconfig编辑器配置
.eslintignore 忽略语法检查的目录文件配置
.eslintrc.jseslint的配置文件
.gitignore 配置git仓库的忽略
index.html项目根视图,入口模板文件
package.json 项目配置文件,定义了项目的相关信息及项目的相关包依赖,npm运行命令等
.postcssrc.js:postcss配置文件
static: 静态文件目录
src/router/index.js: 路由文件,配置路由
src/main.js是入口文件，主要作用是初始化vue实例并使用需要的插件,在main.js文件中定义了一个vue对象,其中el为实例提供挂载元素
/config/index.js: 配置autoOpenBrowser:true设置在浏览器自动打开

模板语法:
vue组件:template script style
