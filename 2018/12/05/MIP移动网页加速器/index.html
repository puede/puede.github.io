MIP移动网页加速器

发表于 2018-12-05 | 更新于 2019-01-03

1.mip介绍

MIP（Mobile Instant Pages - 移动网页加速器），是一套应用于移动网页的开放性技术标准。通过提供 MIP-HTML 规范、MIP-JS 运行环境(可以保证 MIP-HTML 页面的快速渲染)以及 MIP-Cache 页面缓存系统，实现移动网页加速。

这篇文档将带你快速创建一个 MIP 页面。

why mip?
使用<link rel="canonical" href="yourpath.html">能使mip指向原始页面，以保证 MIP 更好的继承原始页面的权重。

2.mip使用规范及组件

先看一个基本mip文件

注意:

  1. 在 <html> 标签中增加 mip 属性标识。
  2. 编码为 utf-8 。
  3. 添加 meta-viewport，用于移动端展现
  4. 添加 MIP 运行环境,即MIP 依赖的 mip.js 和 mip.css
  5. 添加 MIP 关联标签
    <link rel="miphtml">和<link rel="canonical">主要用于告知搜索引擎页面间的关系。添加关联标签后，MIP页的会继承原页面(移动端或PC端) 的点击权重，同时MIP页将作为搜索引擎的首选导流页面。<link rel="miphtml"> 在移动端页面或PC端使用，指向对应内容的 MIP 页，方便搜索引擎发现对应的 MIP 页。<link rel="canonical"> 在 MIP 页中使用，指向内容对应的移动端或PC端。若直接在原链接修改 MIP，则 Canonical 指向当前 URL 。
  6. css样式,出于速度考虑，建议內联使用 CSS 样式。所有样式写在 <style mip-custom></style> 中，style 标签仅允许出现一次。
  7. 注意HTML标签的使用,因为MIP十分关注页面速度，也因此禁用了一些引起拖慢速度的 HTML 标签,如<img/>—><mip-img></mip-img>
  8. 使用组件,在一个合法的 MIP 页面中，所有的交互通过引入 MIP 组件实现。MIP 组件可以理解为封装了 JS 的自定义 HTML 标签
  9. 预览页面,首先使用MIP代码校验工具:https://www.mipengine.org/validator/validate
    保证代码的规范性,然后可以像预览html一样直接打开,也可以本地部署一个服务,或者用MIP-CLI辅助预览

3.MIP-CLI工具安装及环境部署

mip开发工具，用于 MIP 页面和组件的开发和校验

  1. 使用npm/cnpm安装，依赖node环境,开发组件使用git
  2. Node.js (>=4.x) node -v
  3. npm install -g mip-cli(cnpm install -g mip-cli)
  4. mip -V
  5. 文件结构,在本地创建一个开发用的文件夹mip-project，在文件夹中新建 html，mip-ext 两个空文件夹,html 文件夹用来存放后续开发的 mip 页面，mip-ext 用于存放个性化组件。
    mkdir mip-project && cd mip-project && mkdir html && mkdir mip-ext && cd ../
  6. html目录下进行MIP初始化 mip init
  7. 新建一个 MIP 网页 mip add index.html,加组件: mip add index.html mip-share
  8. 校验 MIP 网页: mip validate index.html
  9. 调试 MIP 网页,进入到 html 项目目录,启动 mip server
